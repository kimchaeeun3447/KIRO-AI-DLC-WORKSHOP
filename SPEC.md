# StarGazer — 프로젝트 명세 (SPEC)

> **"오늘 밤 우리 동네에서 어떤 별자리가 보일까?"**
> 위치 / 시간 / 광공해 조건을 입력하면 그 시점에 실제로 보이는 별과 별자리를 시각화하고 추천하는 단일 페이지 웹 앱.
>
> 본 문서는 **요구사항 (WHAT / WHY)** 을 정의한다. 구현 디테일 (색상 hex, CSS 클래스, 알고리즘 의사코드 등) 은 코드가 SOT.

---

## 1. 개요

- **목적**: 관측 위치 · 시간 · 광공해 조건에 따라 실제 보이는 별자리를 계산하여 시각화하고 추천
- **사용자**: 별 보는 일반인, 천체관측 동호인, 여행자
- **핵심 가치**: 천문 계산 정확성 + 직관적 시각화 + 빠른 응답
- **배포 모델**: localhost only (단일 사용자, 워크샵 데모)
- **언어**: 한국어 UI 전용

## 3. 데이터

### `data/stars.csv` (~9,000 행)

| 컬럼 | 설명 |
|---|---|
| `hip` | Hipparcos ID (0 = 미지정) |
| `ra`, `dec` | 적경 (시간) / 적위 (도) |
| `mag` | 겉보기 등급 (≤ 6.5 까지 수록) |
| `spect` | 분광형 |
| `con` | 별자리 IAU 3글자 약어 |
| `proper` | 고유명 (있을 때만 — Sirius, Vega 등) |
| `bayer` | 바이어 명칭 (있을 때만) |
| `dist` | 거리 (파섹) |

### `data/constellations.json` (88개)

각 항목: `code` (IAU 약어) · `name_ko` / `name_en` / `name_la` · `line_pairs` (HIP 쌍 배열) · `myth` (한국어 설명).

`line_pairs` 의 HIP 을 stars.csv 와 조인해 별자리 선을 그린다.

## 4. 천문 계산

### 4.1 필요한 계산

다음을 산출:

- 줄리안 날짜 (JD)
- 항성시 (LST)
- 적도 → 지평 좌표 변환 (Alt / Az)
- 태양 좌표 (적경 / 적위)
- 일몰 / 일출 / 시민 박명 / 천문 박명 시각
- 달 위상 + illumination + 한국어 위상 이름 / 이모지

### 4.2 구현 방식 (자유 선택)

다음 중 하나:

- **AstroPy / astroplan 사용** (권장 — 정확하고 간결)
- **직접 구현** — 출처: USNO Almanac for Computers (1990) 의 Ed Williams 박명 알고리즘, USNO "Computing Approximate Solar Coordinates"

정확도는 § 9 NFR 와 § 12 검증 시나리오로 검증한다.

### 4.3 한국어 달 위상 이름 매핑

(`phase` 는 0~1, 0 = 신월 / 0.5 = 보름)

| phase 범위 | 이름 | 이모지 |
|-----------|------|--------|
| 0.00 ~ 0.03 | 신월 | 🌑 |
| 0.03 ~ 0.25 | 초승달 | 🌒 |
| 0.25 ~ 0.28 | 상현달 | 🌓 |
| 0.28 ~ 0.47 | 차오르는 달 | 🌔 |
| 0.47 ~ 0.53 | 보름달 | 🌕 |
| 0.53 ~ 0.72 | 기우는 달 | 🌖 |
| 0.72 ~ 0.75 | 하현달 | 🌗 |
| 0.75 ~ 0.97 | 그믐달 | 🌘 |
| 0.97 ~ 1.00 | 신월 | 🌑 |

## 6. 기능 요구사항

### FR-1 입력

- **위치**: 사전 정의 도시 8곳 드롭다운 + 위경도 직접 입력 + 도시 선택 시 기본 Bortle 자동 적용
- **날짜 / 시간**: 기본값 = 현재 (KST). 백엔드 호출 시 UTC ISO 8601 로 변환
- **광공해**: Bortle 1~9 슬라이더 + 환경 설명 한국어 표시 + 한계 등급 표시

**사전 정의 도시:**

| 도시 | 위도 | 경도 | 기본 Bortle |
|------|------|------|------------|
| 서울 강남 | 37.498 | 127.028 | 8 |
| 서울 종로 | 37.572 | 126.977 | 8 |
| 부산 해운대 | 35.163 | 129.160 | 7 |
| 대전 | 36.351 | 127.385 | 7 |
| 제주 | 33.499 | 126.531 | 5 |
| 영월 | 37.184 | 128.462 | 3 |
| 양양 | 38.075 | 128.619 | 3 |
| 무주 | 35.922 | 127.661 | 3 |

기본 위치 = 서울 강남.

### FR-2 천문 계산 (백엔드 내부)

- § 4 의 모든 계산 산출
- 8,920 별 전체에 대한 Alt / Az 일괄 계산이 NFR-1 안에 들어와야 함

### FR-3 가시성 필터링

- 지평선 위 (Alt > 0°) 별만 표시
- Bortle 한계 등급 (§ 5) 적용
- 별자리 내 가시 별이 **2개 이상** 일 때만 별자리 표시
- 별자리 모양 보존을 위해, 별자리 표시 조건 통과 시 line_pairs 의 양 끝이 지평선 위에 있으면 mag 한계와 무관하게 선을 그린다 (점은 mag 한계 적용)

### FR-4 별자리 추천 점수

```
score = 0.4 * visibility_pct
      + 0.35 * altitude_score   # 평균 고도 30°~80° 사이에서 최고
      + 0.25 * star_count_score
```

- 가중치는 위 default 사용
- Top 3 추출 → 응답에 포함
- UI 노출 시 0~10 스케일 (소수점 1자리)

### FR-5 시각화 (SkyMap)

- 원형 지평선 돔 SVG (방위각 → 각도, 고도 → 중심으로부터 거리)
- 별 등급에 따라 점 크기 시각적으로 구분 (밝을수록 큼, 3단계)
- 별자리 선 / 한국어 라벨 / 방위 표시 (N / S / E / W) / 고도 보조선 (30° / 60°)
- **확대 / 축소**: 마우스 휠 (커서 위치 기준), +/− 버튼, 리셋 버튼. 줌 1× 초과 시 드래그로 panning
- 좌표 변환은 표준 천체 돔 투영 (zenith 중심)

### FR-6 별자리 상세 정보

- 별자리 클릭 시 (SkyMap 선/라벨 또는 Top 3 카드) **풀폭 하단 드로어**가 등장
- 좌측: 선택된 별자리만 isolated 확대 보여주는 미니 맵 (모양 인식)
- 우측: 한국어 / 라틴어 / IAU 약어, 평균 고도, 추천 점수, 신화 텍스트, 소속 별 목록 (mag 정렬, 더 보기 / 접기 토글로 전체 표시)
- 닫기: X 버튼 / 같은 별자리 재클릭 / Escape 키
- 모바일에서는 좌우가 위/아래로 stack

### FR-7 시간 슬라이더 (P4)

- 일몰 ~ 다음날 일출 범위
- 천문 박명 종료 ~ 시작 구간을 트랙에 시각적으로 강조
- 자정 횡단 시에도 슬라이더 범위가 안정 (사용자가 선택한 관측 야간 기준 고정)
- 백야 / 극야 시 비활성 + 메시지

### FR-8 자동 갱신

- 입력 변경 시 ~300 ms 디바운스 후 자동으로 백엔드 재호출
- 명시적 "관측 시작" 버튼 없음

### FR-9 에러 처리

- HTTP 4xx / 5xx / 네트워크 실패 → 한국어 토스트 메시지
- 토스트는 자동 사라짐 + 수동 닫기
- 마지막 정상 응답으로 그려진 화면은 그대로 유지 (빈 화면 금지)
- 입력 검증 실패 (위경도 범위 등) 는 API 호출 보류 + 토스트로 안내

### FR-10 API 계약

§ 7 응답 스키마 그대로 준수.

**Request**
```json

  "longitude": 127.028,
  "datetime_utc": "2026-06-06T12:30:00Z",
  "bortle_class": 7
}
```

사전 정의 도시 8곳 반환. 각 항목: `key` / `name` / `lat` / `lon` / `default_bortle`.

### `GET /api/constellations/{code}`

코드별 상세: `code` / `name_ko` / `name_en` / `name_la` / `myth` / `stars[]` (각 별: `hip` / `proper` / `bayer` / `mag`, mag 오름차순).

미발견 시 404.

## 8. UI / UX 요구사항

### 8.1 레이아웃 의도

```
Header (로고 + 위치 + 현재 시각)
Controls (위치 / 위경도 / 날짜 / 시간 / Bortle)
TimeSlider (일몰 ~ 일출, 박명 강조)
─────────────────────────────────
SkyMap (좌, 메인)        │ InfoPanel  (일몰/박명/달)
                         │ Top3Panel  (Top 3 별자리)
─────────────────────────────────
ConstellationDetail (선택 시 풀폭 하단, 미니 맵 + 신화 + 소속 별)
─────────────────────────────────
Toast (에러 시 상단 중앙)
```

모바일 (< 768px) 은 1열 세로 stack.

### 8.2 디자인 의도

코드 / 코드 코멘트가 정확한 hex / class / animation 의 SOT. 본 절은 의도만 묘사한다.

- **분위기**: 깊은 밤하늘 + 떠다니는 오로라. 은은한 인디고 / 바이올렛 / 시안의 그라데이션이 배경에 깔리고, 위에 큰 오로라 블롭이 천천히 흐름. 멀리 미세한 별 분포.
- **패널**: 글래스 모피즘 — 반투명 슬레이트 배경 + 백드롭 블러 + 미세한 흰 보더로 분리감. 모서리 둥글게.
- **별 점**: 흰계열, 등급에 따라 크기 차등 (3단계). 부드러운 발광 효과. 가장 밝은 별만 천천히 깜빡임 (수 초 주기).
- **별자리 선**: 옅은 흰색. 선택된 별자리는 시안 → 앰버 → 핑크의 오로라 그라데이션으로 빛남.
- **별자리 라벨**: 옅은 흰계열, 선택 시 따뜻한 앰버.
- **강조 텍스트** (관측 시작 / 종료 시각, 점수, 한계 등급, 슬라이더 시각): 따뜻한 앰버 계열.
- **그라데이션 텍스트** (로고, 패널 제목): 시안 → 바이올렛 → 핑크.
- **TimeSlider 트랙**: 일몰 (앰버) → 박명 (인디고) → 천문 박명 사이 (검정) → 박명 → 일출 (앰버) 의 다단계 그라데이션.
- **에러 토스트**: 로즈 계열 글래스 카드 + 부드러운 발광.
- **헤더 디바이더**: 1px 가로 오로라 그라데이션 라인.

### 8.3 인터랙션 / 모션 의도

- 입력 변경 → 디바운스 후 자동 갱신 (FR-8)
- SkyMap 줌: 마우스 휠은 **커서 위치를 앵커로**, 드래그로 panning (FR-5)
- ConstellationDetail: 슬라이드 업으로 부드럽게 등장
- 인터랙티브 요소: 짧은 transition (~150 ms) 으로 색 전환
- 가장 밝은 별: 부드러운 깜빡임 (수 초 주기)
- 달 이모지: 호흡하듯 발광
- 오로라 블롭: 수십 초 주기로 매우 천천히 떠다님
- **사용자가 `prefers-reduced-motion` 를 선호하면 모든 비-필수 애니메이션 비활성**

### 8.4 반응형 / 접근성

- 최소 가로 375 px, < 768 px 1열 stack
- 터치 타겟 최소 44 × 44 px
- WCAG 2.1 AA 대비비
- 키보드 네비게이션 (Tab / Enter / Space / Escape)
- SVG 에 `<title>` + `<desc>`, 인터랙티브 요소에 `aria-label`
- `<html lang="ko">`
